📖 Orientações de Envio de Newsletters

1. Providers de E-mail

Atualmente usamos Vercel para testes. Para produção, basta alterar EMAIL_PROVIDER para "ses" e configurar o endpoint /api/sesEnviarEmail.

2. Variáveis de Ambiente

  • AWS_ACCESS_KEY_ID
  • AWS_SECRET_ACCESS_KEY
  • AWS_REGION
  • SES_SOURCE_EMAIL (e-mail verificado no SES)
  • SES_REPLYTO_EMAIL (opcional)

Configure essas variáveis no painel do Vercel (para testes) ou no ambiente AWS (produção).

3. Regras de Envio

  • Leads → apenas newsletters básicas, se receber_newsletter !== false.
  • Usuários → podem receber premium se assinatura estiver ativo.
  • Canceladas/expiradas não recebem.

4. Rate Limiting e Paralelismo

O envio usa pool de workers com limite de paralelismo e pausa entre envios. Exemplo atual: 5 workers + 600ms delay → ~50–100/min.

5. Checklist de Testes

  • Prévia mostra apenas destinatários válidos.
  • Botão "Preparar envio" só habilita após escolher tipo de destinatário.
  • Envio registra status no Firestore (enviado ou erro).
  • Barra de progresso atualiza corretamente.
